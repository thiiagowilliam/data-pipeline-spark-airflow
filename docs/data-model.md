# Data Model and Schemas

This document describes the data models, schemas, and data contracts used in the pipeline.

## Raw Data (CSV)

The raw data is generated by `simulator/data_simulator.py` in CSV format.

### `clientes_*.csv`

| Column | Type | Description |
|--------|------|-------------|
| `id` | integer | Unique customer identifier |
| `nome` | string | Customer name |
| `email` | string | Email address |
| `telefone` | string | Phone number |
| `cidade` | string | City |
| `estado` | string | State abbreviation (2 chars, e.g. `SP`) |
| `data_cadastro` | date | Registration date |
| `status` | string | Account status (`Ativo`, `Inativo`, `Pendente`) |

### `vendas_*.csv`

| Column | Type | Description |
|--------|------|-------------|
| `id` | integer | Unique sale identifier |
| `cliente_id` | integer | Foreign key → `clientes.id` |
| `produto_id` | integer | Product identifier |
| `data_venda` | date | Sale date |
| `valor_total` | float | Total value (must be > 0) |
| `quantidade` | integer | Quantity sold (must be ≥ 1) |
| `metodo_pagto` | string | Payment method (`PIX`, `Boleto`, `Cartão de Crédito`, `Dinheiro`) |

## Bronze Layer (Delta Lake)

Stored at `s3a://datalake/bronze/{dataset}/{file_id}` in Delta Lake format. Schema matches the raw CSV data with Spark-inferred types. Provides ACID transactions and schema enforcement.

## PostgreSQL Data Warehouse

### `clientes`

```sql
CREATE TABLE clientes (
    id          INTEGER PRIMARY KEY,
    nome        VARCHAR(255),
    email       VARCHAR(255),
    telefone    VARCHAR(50),
    cidade      VARCHAR(100),
    estado      VARCHAR(50),
    data_cadastro DATE,
    status      VARCHAR(50)
);
```

### `vendas`

```sql
CREATE TABLE vendas (
    id          INTEGER PRIMARY KEY,
    cliente_id  INTEGER REFERENCES clientes(id),
    produto_id  INTEGER,
    data_venda  DATE,
    valor_total FLOAT,
    quantidade  INTEGER,
    metodo_pagto VARCHAR(50)
);
```

### `airflow_file_metadata`

```sql
CREATE TABLE airflow_file_metadata (
    id           SERIAL PRIMARY KEY,
    file_key     VARCHAR(500) UNIQUE NOT NULL,
    process_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status       VARCHAR(50) NOT NULL
);
```

## Data Contracts

JSON contract files in `contracts/` define the expected schema for each dataset. These are loaded by the `DataValidator` class at runtime.

| Contract | Dataset | Validations |
|----------|---------|-------------|
| `clientes.json` | Clientes | 8 columns, types, nullable rules |
| `vendas.json` | Vendas | 7 columns, types, nullable rules |
| `sales_contract.json` | Vendas (extended) | GE rules: not_null, type checks, value ranges, valid product set |

### Business Rules (validated at runtime)

| Dataset | Rule | Description |
|---------|------|-------------|
| Clientes | `email_format` | Must match email regex |
| Clientes | `state_length` | `estado` must be exactly 2 characters |
| Clientes | `valid_status` | Must be in `{Ativo, Inativo, Pendente}` |
| Vendas | `positive_value` | `valor_total` must be > 0 |
| Vendas | `min_quantity` | `quantidade` must be ≥ 1 |
| Vendas | `valid_payment` | `metodo_pagto` must be in valid set |
| Vendas | `unique_id` | `id` should be unique (warning) |

## Great Expectations Suites

Pre-configured expectation suites in `great_expectations/expectations/`:

- **`clientes_suite.json`**: Column order, not_null (id, email, estado), id type, email regex, estado length, valid status
- **`vendas_suite.json`**: Column order, not_null (id, cliente_id), id/cliente_id/quantidade type, valor_total range
